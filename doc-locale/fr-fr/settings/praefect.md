[Gitaly Cluster (Praefect)](https://docs.gitlab.com/administration/gitaly/praefect/) fournit un stockage tolérant aux pannes pour les dépôts. Il utilise Praefect comme routeur et gestionnaire de transactions pour Gitaly.

## Activer Gitaly Cluster (Praefect) {#enable-gitaly-cluster-praefect}

Par défaut, Gitaly Cluster (Praefect) n'est pas activé. Pour en savoir plus sur l'activation de Gitaly Cluster (Praefect), consultez les [instructions de configuration](https://docs.gitlab.com/administration/gitaly/praefect/configure/#setup-instructions) de Gitaly Cluster (Praefect).

## Mettre à jour GitLab lorsque Gitaly Cluster (Praefect) est activé {#update-gitlab-when-gitaly-cluster-praefect-is-enabled}

Pour en savoir plus sur la mise à jour de GitLab avec Gitaly Cluster (Praefect) activé, consultez les [instructions spécifiques](https://docs.gitlab.com/update/zero_downtime/#upgrade-gitaly-cluster-praefect-nodes).
